## 程式碼範例

### 1. 取得特定經卷的向量資料庫

```javascript
// 在 /api/bible/commentary/stream 中的實作
const storePrefix = process.env.BIBLE_STORE_PREFIX || 'Bible-';
const targetName = `${storePrefix}${bookEn}`;
const storeResult = await getVectorStoreIdCachedByName(targetName);

if (!storeResult) {
  return res.status(503).json({ 
    success: false, 
    error: `該卷資料庫尚未建立完成，請稍後再試（${targetName}）` 
  });
}
```

### 2. 使用向量資料庫進行工具註釋

```javascript
// 創建工具 Assistant 時指定向量資料庫
const assistant = await getOrCreateBibleCommentaryAssistant(targetVectorStoreId);

// 在串流 Run 中覆寫 tool_resources
const stream = await openai.beta.threads.runs.stream(thread.id, {
  assistant_id: assistant.id,
  tool_resources: {
    file_search: { vector_store_ids: [targetVectorStoreId] }
  }
});
```

### 3. 前端呼叫範例

```javascript
const params = {
  bookEn: 'Genesis',    // 對應 Bible-Genesis
  ref: '6:4',
  translation: 'CUV',
  passageText: '那時候有偉人在地上...'
};

await processCommentaryToolStream('/api/bible/commentary/stream', params, commentaryDiv);
```

## 資料庫內容

每個向量資料庫包含：
- 該經卷的神學註釋文獻
- 經文解釋和詮釋資料
- 歷史背景和語言學資料
- 神學家和聖經學者的評論

## 注意事項

1. 資料庫名稱區分大小寫
2. 空格和連字符必須完全匹配
3. 某些經卷可能尚未建立完成（file_counts.total = 0）
4. 建議檢查資料庫可用性後再進行查詢
5. 向量資料庫 ID 格式：vs_xxxxxxxxxxxxxxxxxxxxxxxx

## 環境變數

- `BIBLE_STORE_PREFIX`: 資料庫名稱前綴（預設: "Bible-"）
- `VECTOR_STORE_ID`: 主要向量資料庫 ID（用於檔案分類）
- `OPENAI_API_KEY`: OpenAI API 金鑰
